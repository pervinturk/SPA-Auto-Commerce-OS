## 🎯 Problem: E-Ticaretin Görünmeyen Karar Felci

Türkiye’deki 500.000’den fazla aktif e-ticaret satıcısının %78'i operasyonel ve finansal kararlarını hâlâ manuel süreçlerle, sezgisel olarak alıyor. Bu durum şu yıkıcı krizlere yol açıyor:

- ⏱ **Operasyonel Darboğazlar** — Manuel envanter girişi, tedarik planlama ve fiyatlandırma hataları yüzünden kaybedilen binlerce iş saati.
- 📉 **Görünmez Marj Erozyonu** — Optimize edilmeyen kargo süreçleri, dinamik lot büyüklüklerinin hesaplanamaması ve tedarikçi gecikmeleri nedeniyle eriyen kâr marjları.
- 🔀 **Veri Siloları ve Finansal Körlük** — Pazar yeri API'leri, lojistik logları ve döviz dalgalanmalarının tek bir analitik düzlemde birleşememesi yüzünden oluşan nakit akışı riskleri.

### 🤖 ReAct Agent — Akıl Hocası & Human-in-the-Loop
Sistemin merkezinde **ReAct (Reasoning + Acting)** döngüsü koşturan otonom bir akıl hocası bulunur. Doğal dildeki kullanıcı taleplerini ya da sistem alarmlarını analiz ederek arka plandaki SQL araçlarını ve Yöneylem çözücülerini tetikler. Fiyat güncelleme, tedarik emri açma veya kargo birleştirme gibi kritik operasyonel eylemlerde **Human-in-the-Loop** prensibini işleterek kullanıcıdan tek tıkla onay alır ve döngüyü kapatır.

### 🛡 Üç Katmanlı Halüsinasyon Güvenlik Duvarı (Safety Layer)
Finansal ve operasyonel hataları sıfıra indirmek amacıyla, LLM çıktılarının doğrudan sisteme etki etmesi yazılımsal olarak engellenmiştir:
1. **Üretim Fazı (AI Propose):** Gemini 1.5 Flash ham karar veya payload taslağını üretir.
2. **Validasyon Fazı (Strict Schema):** Üretilen veri Pydantic şemalarından geçer, deterministik kural motoru ve çapraz veri kontrolleriyle doğrulanır.
3. **Uygulama Fazı (Safe Action):** Matematiksel olarak doğruluğu kanıtlanan aksiyon kullanıcı onayına sunulur.

<img width="2400" height="1350" alt="6_Hallucination-Safety-Layer-Guvenilir-AI" src="https://github.com/user-attachments/assets/1a3e9ffc-33c6-41b9-9130-1a847ba353cc" />

### 📊 Endüstri Mühendisliği ve Lojistik Optimizasyon Çekirdeği
Sistemimiz yapay zeka sarmalayıcılarının ötesine geçerek, arka planda saf mühendislik algoritmaları çalıştırır:

| Matematiksel Model / Algoritma | Sistemdeki Operasyonel Karar Alanı |
| :--- | :--- |
| **Wagner-Whitin (Dinamik Programlama)** | Değişen talep matrislerinde envanter elde tutma ve sipariş kurulum maliyetlerini minimize eden lot büyüklüğü optimizasyonu. |
| **TOPSIS (Çok Kriterli Karar Verme)** | Kargo firmalarını; maliyet, teslim süresi (ETA), hasar oranı ve SLA başarılarına göre matris düzeyinde ağırlıklandırarak otonom seçme. |
| **Parametrik VaR (Value at Risk)** | Acklam invers normal CDF yaklaşımıyla döviz pozisyonlarındaki maksimum finansal riski hesaplama ve otonom hedge önerileri üretme. |
| **Damerau-Levenshtein Metriği** | Süreç Madenciliği (Process Mining) kapsamında sipariş adımlarındaki darboğazları ve operasyonel sıra sapmalarını (Deviation) algılama. |
| **(s, Q) Stok Politikası & EOQ** | Emniyet stoğu, reorder point (ROP) ve ekonomik sipariş miktarının analitik olarak belirlenmesi. |

$$\text{EOQ} = \sqrt{\frac{2DS}{H}} \quad , \quad \text{ROP} = (\text{Daily Demand} \times \text{Lead Time}) + \text{Safety Stock}$$

## 🛠 Teknoloji Yığını (Tech Stack)

| Katman | Teknoloji | Neden Seçildi |
|--------|-----------|---------------|
| **AI/LLM** | Google Gemini 1.5 Flash | Hız, maliyet etkinliği, multimodal destek |
| **Agentic** | ReAct pattern + Function Calling | Deterministik + LLM hibrit kontrol |
| **UI** | CustomTkinter + Matplotlib | Düşük gecikme, masaüstü OS hissi |
| **DB** | SQLite (WAL Mode) + SQLAlchemy Async | Yüksek eşzamanlılık, yerel güvenlik |
| **Validasyon** | Pydantic (Strict Schema) | Halüsinasyon koruması, tip güvenliği |
| **Optimizasyon** | NumPy + SciPy | Matematiksel modeller için endüstri standardı |

## ⚙️ Kurulum ve Çalıştırma

1.  **Projeyi Klonlayın ve Klasöre Geçin:**
    ```bash
    git clone [https://github.com/TEAM-EXCEL/spa-autocommerce-os.git](https://github.com/TEAM-EXCEL/spa-autocommerce-os.git)
    cd spa-autocommerce-os
    ```
2.  **Gerekli Bağımlılıkları Yükleyin:**
    ```bash
    pip install -r requirements.txt
    ```
3.  **Ortam Değişkenlerini Yapılandırın:**
    `""` klasöründe bir `.env` dosyası oluşturun ve Gemini API anahtarınızı tanımlayın:
    ```env
    GEMINI_API_KEY=your_google_gemini_api_key_here
    ```
4.  **Uygulamayı Başlatın:**
    ```bash
    python main_gui.py
    ```
